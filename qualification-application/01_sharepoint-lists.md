# SharePoint Lists 設計・作成手順

## 1. サイト準備

### 1.1 SharePointサイトにアクセス
1. https://[テナント名].sharepoint.com にアクセス
2. 既存のチームサイトを使用、または新規作成
   - 「+ サイトの作成」→「チームサイト」
   - サイト名: 例「社内申請システム」

---

## 2. 資格マスタリスト作成

### 2.1 リスト作成
1. サイトのホームページで「+ 新規」→「リスト」
2. 「空白のリスト」を選択
3. 名前: `資格マスタ`
4. 「作成」をクリック

### 2.2 列の追加

| 列名 | 種類 | 設定詳細 |
|-----|------|---------|
| Title | 1行テキスト | 既存列を「資格名」に名前変更 |
| 分類 | 選択肢 | 下記参照 |
| 詳細 | 1行テキスト | 級・レベル等 |
| 報奨金 | 数値 | 小数点以下0桁、通貨記号なし |
| 難易度 | 選択肢 | S/A/B/C/D |
| 参考URL | ハイパーリンク | 任意 |

#### 分類の選択肢
```
税務
会計
お金
労務社保
給与
IT
その他
```

### 2.3 列の追加手順（詳細）

1. リスト画面右上「+ 列の追加」
2. 「選択肢」を選択
3. 名前: `分類`
4. 選択肢を入力（上記参照）
5. 「保存」

同様に他の列も追加

## 3. 申請リスト作成

### 3.1 リスト作成
1. 「+ 新規」→「リスト」→「空白のリスト」
2. 名前: `資格取得申請`
3. 「作成」

### 3.2 列の追加

| 列名 | 種類 | 設定詳細 |
|-----|------|---------|
| Title | 1行テキスト | 「申請ID」に名前変更（自動採番用） |
| 申請種別 | 選択肢 | 宣言届 / 取得届 |
| 申請者 | ユーザー | 1人のみ選択可 |
| 申請日 | 日付と時刻 | 日付のみ |
| 取得資格 | 参照 | 資格マスタの「資格名」を参照 |
| 資格詳細 | 参照 | 資格マスタの「詳細」を参照 |
| 予定日_取得日 | 日付と時刻 | 日付のみ |
| 理由_内容 | 複数行テキスト | プレーンテキスト |
| 備考 | 複数行テキスト | プレーンテキスト |
| ステータス | 選択肢 | 申請中 / 承認済 / 却下 |
| 承認者 | ユーザー | 1人のみ選択可 |
| 承認日 | 日付と時刻 | 日付のみ |
| 事前申請フラグ | はい/いいえ | 既定値: いいえ |
| 報奨金額 | 数値 | 計算用 |
| 事前ボーナス | 数値 | +5000円分 |

### 3.3 参照列の設定手順

1. 「+ 列の追加」→「参照」
2. 名前: `取得資格`
3. 情報の取得元: `資格マスタ`
4. この列に表示: `資格名`（Title）
5. 「保存」

### 3.4 ステータスの選択肢
```
申請中
承認済
却下
```

### 3.5 ビューの設定

#### 申請一覧ビュー
1. リスト画面右上「すべてのアイテム」→「ビューの作成」
2. 名前: `申請一覧`
3. 表示する列:
   - 申請ID
   - 申請種別
   - 申請者
   - 取得資格
   - 予定日_取得日
   - ステータス
4. 並び替え: 申請日（降順）

#### 承認待ちビュー
1. ビューの作成
2. 名前: `承認待ち`
3. フィルター: ステータス = 申請中

## 4. 権限設定

### 4.1 リストの権限
1. リスト設定 → 「このリストに対する権限」
2. 権限の継承を停止
3. 設定:
   - 全従業員: 投稿（自分のアイテムのみ編集可）
   - 承認者グループ: 編集
   - 管理者: フルコントロール

### 4.2 承認者グループ作成
1. サイトの設定 → 「サイトの権限」
2. 「グループの作成」
3. 名前: `資格申請承認者`
4. 承認者を追加

---

## 5. 確認事項

- [ ] 資格マスタにデータが登録されている
- [ ] 申請リストの参照列が正しく資格マスタを参照している
- [ ] ステータスの既定値が「申請中」になっている
- [ ] 権限設定が完了している
